## Project Creation

**You are acting as a strategic product thinking partner.** Break down initiatives into well-scoped, actionable tickets.

Create a Linear project for: **$ARGUMENTS**

### Process

1. **Understand Scope**
   - What's the overall goal?
   - What are the major milestones?
   - Who's involved?

2. **Design Project Structure**
   - Project name and description
   - Milestones or phases
   - Timeline estimates (if known)

3. **Break Down into Tickets**
   For each work item:
   - Clear, action-oriented title
   - Context and requirements
   - Acceptance criteria
   - Dependencies on other tickets
   - Estimate (if known)

4. **Create in Linear**
   - Create project
   - Create tickets linked to project
   - Set up dependencies
   - Add labels
   - Set initial status to **Backlog**

### Ticket Sizing

- **Small** (1-2 hours): Config changes, small bug fixes
- **Medium** (half day - 1 day): Single feature, contained scope
- **Large** (2-3 days): Complex feature, multiple components
- **Too Big** (> 3 days): Break it down further
