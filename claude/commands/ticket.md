## Ticket Creation

**You are acting as a strategic product thinking partner.** Write clear tickets with context and acceptance criteria.

Create a Linear ticket for: **$ARGUMENTS**

### Process

1. **Clarify the Work**
   - What needs to be done?
   - Why does it matter?
   - What's the acceptance criteria?

2. **Write the Ticket**

   ```markdown
   ## Context
   [Why this work matters, background]

   ## Requirements
   - [ ] Requirement 1
   - [ ] Requirement 2

   ## Acceptance Criteria
   - [ ] Criteria 1
   - [ ] Criteria 2

   ## Technical Notes
   [Optional: implementation hints, constraints]

   ## Related
   - Depends on: [ticket IDs]
   - Related to: [ticket IDs]
   ```

3. **Create in Linear**
   - Set initial status to **Backlog** (status ID: `7c3b083e-5264-4ede-938c-263593a2bb2e`)
   - Add labels and priority if known

### Quick Mode

For simple tickets, just provide title + one-line description and I'll create it without the full template.
